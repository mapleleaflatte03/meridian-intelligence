Open-source onboarding path

Get Meridian running locally in one flow.

This page replaces the old pilot checkout surface. Meridian is now contribution-first and open-source. Use this path to run Loom + Kernel + Intelligence on your own host.

Scope
What this gives you Full local stack with governed runtime surfaces.
What this is not No purchase flow, no license checkout, no gated usage.

Support channels stay voluntary and separate from onboarding.

1. Clone the monorepo

git clone https://github.com/mapleleaflatte03/meridian.git
cd meridian

2. Bootstrap full stack

curl -fsSL https://raw.githubusercontent.com/mapleleaflatte03/meridian/main/scripts/install-full.sh | bash

This one command clones/reuses the monorepo, initializes kernel state, and builds the Loom CLI in release mode.

3. Run gateway and inspect live surfaces

cd intelligence
python3 meridian_gateway.py
  • Open /proofs for runtime proof posture.
  • Open /workflows for live workflow showcase.
  • Open /boundary for current claim boundary.

4. Create and run your first agent

export LOOM_ROOT="${HOME}/.local/share/meridian-loom/runtime/default"
export MERIDIAN_ROOT="$(pwd)"
export MERIDIAN_KERNEL_PATH="${MERIDIAN_ROOT}/kernel"
export MERIDIAN_ORG_ID="${MERIDIAN_ORG_ID:-local_foundry}"

cd "${MERIDIAN_KERNEL_PATH}"
python3 quickstart.py --init-only

cd "${MERIDIAN_ROOT}/loom"
./target/release/loom new-agent \
  --name "My Assistant" \
  --root "$LOOM_ROOT" \
  --kernel-path "$MERIDIAN_KERNEL_PATH" \
  --org-id "$MERIDIAN_ORG_ID"

./target/release/loom run-agent my-assistant

5. Contribute back

If you find gaps, contribute tests/docs/patches:
